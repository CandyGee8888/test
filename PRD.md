---

## 1. 背景與目標

### 1.1 現況問題
目前版本的俄羅斯方塊遊戲存在以下兩個主要問題，導致使用者體驗不佳：
1.  **遊戲流程中斷**：現有代碼在方塊落地後，無法正確生成下一個方塊或遊戲邏輯出錯，導致遊戲無法連續進行（落一個就完了）。
2.  **設備兼容性不足**：缺乏螢幕虛擬按鍵，導致在 iPad 或其他觸控設備上無法操作（僅支援鍵盤操作）。

### 1.2 專案目標
*   **修復核心邏輯**：實現方塊落地後自動無限循環生成下一個方塊，確保遊戲可以持續進行直到 Game Over。
*   **增加觸控支援**：新增螢幕虛擬控制器，包含方向鍵與功能鍵，使遊戲可在 iPad、手機等觸控設備上流暢遊玩。

---

## 2. 功能需求

### 2.1 遊戲核心邏輯優化
**需求 ID**: REQ-001
**描述**: 修復方塊生成機制，確保遊戲循環。
**具體細節**:
*   當前方塊碰撞並固化到棋盤後，系統必須立即呼叫「生成新方塊」函數。
*   新生成的方塊應來自「預覽區」，同時預覽區需立即刷新生成新的預覽方塊。
*   若新生成的方塊在生成時即發生碰撞，則判定為 Game Over。

### 2.2 虛擬控制器
**需求 ID**: REQ-002
**描述**: 在遊戲主畫面下方增加螢幕按鈕，以支援滑鼠點擊與手指觸控。
**按鈕清單**:
1.  **左移按鈕 (←)**：點擊時方塊向左移動一格。
2.  **右移按鈕 (→)**：點擊時方塊向右移動一格。
3.  **旋轉按鈕 (↑ / ↻)**：點擊時順時針旋轉方塊 90 度。
4.  **加速下落按鈕 (↓)**：點擊時方塊加速向下移動一格（軟掉落）。
5.  **即時落下按鈕**：點擊時方塊直接瞬間掉落到底部並固化（硬掉落）。

## 3. 互動與 UI 設計

### 3.1 佈局調整
*   **PC 端**：保持原有佈局，虛擬按鍵可選顯示或作為輔助。
*   **iPad / 平板**：
    *   畫面需居中顯示。
    *   遊戲棋盤與按鍵區域需調整比例，確保按鍵區域足夠大且不擁擠。
    *   按鍵應採用網格或十字佈局，符合直覺操作。

### 3.2 按鈕視覺規範
*   **尺寸**：按鈕最小觸控區域建議為 44x44 px (Apple HIG 標準) 或更大，以適配手指操作。
*   **回饋**：按鈕在按下時需有明顯的視覺變化（如顏色加深、縮小或陰影變化），提供操作反饋。

---

## 4. 技術需求

### 4.1 事件處理
*   **觸控優化**：虛擬按鈕需綁定 `touchstart` 或 `mousedown` 事件，避免移動端瀏覽器 300ms 的點擊延遲，確保操作跟手。
*   **防止誤觸**：需在 CSS 中加入 `user-select: none` 及 `touch-action: manipulation`，防止連續點擊時觸發縮放或選取文字。

### 4.2 代碼結構調整
*   需重構 `createPiece()` 邏輯，將其拆分為 `generateNextPiece()`（生成緩衝區方塊）與 `spawnPiece()`（將緩衝區方塊放入遊戲區），以解決落一個就停的 Bug。

---

## 5. 驗收標準

1.  **連續性測試**：
    *   [ ] 開始遊戲後，第一個方塊落地，第二個方塊自動生成。
    *   [ ] 消除行數或得分後，遊戲持續進行，不會卡住。
    *   [ ] 直到頂部無空間放置新方塊時，正確顯示 Game Over。

2.  **操作測試 (iPad/手機)**：
    *   [ ] 使用觸控操作，左右移動功能正常。
    *   [ ] 使用觸控操作，旋轉功能正常。
    *   [ ] 點擊「即時落下」按鈕，方塊直接到底且不卡頓。
    *   [ ] 在 Safari/Chrome 瀏覽器中，畫面不會因為點擊過快而縮放或滾動。
